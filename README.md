# QA Testing Project – E-commerce Website

This project demonstrates my QA skills, including manual testing, API testing, and bug reporting for an E-commerce website.

---

## 📝 Project Description
This repository contains:
- **Test Cases** for various user scenarios  
- **Bug Reports** documented with steps, severity, and priority  
- **API Testing** using Postman  

It showcases my ability to ensure software quality, apply ISTQB concepts, and collaborate effectively with developers.

---

## 🛠 Tools Used
- **Postman** – API testing  
- **Jira** – Bug tracking and reporting  
- **Markdown** – Documentation  
- **SQL** – Basic database validation  

---

## 🔹 Test Cases (Examples)
1. **Login Functionality** – Valid & invalid credentials  
2. **User Registration** – Form validation & error handling  
3. **Add to Cart** – Verify products are added correctly  
4. **Checkout Process** – Confirm order placement  

---

## 🐞 Bug Reports (Examples)
- **Bug 1: Login button not working**  
  - Steps to reproduce: Enter valid credentials and click login  
  - Expected: User logs in successfully  
  - Actual: Nothing happens  
  - Severity: High  
  - Priority: High  

- **Bug 2: Cart not updating**  
  - Steps: Add product to cart  
  - Expected: Cart count increases  
  - Actual: Count remains same  
  - Severity: Medium  

---

## 🎯 Goal
The main goal of this project is to practice real-world QA testing scenarios, improve attention to detail, and demonstrate my skills in software testing.
